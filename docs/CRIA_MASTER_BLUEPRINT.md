# CRIA Master Blueprint v1.0

**Convergent Research Intelligence Architecture — Unified Dual-Pipeline System**

*Authoritative architectural specification. All builds reference this document.*

*Author: Dr Barry Ferrier with Claude (Anthropic), 30 April 2026*

---

## 1. System Overview

CRIA is a unified research instrument comprising two architecturally distinct
research pipelines and a comparison engine that operates across both. The
system runs from a single entry point: one research question, one dashboard,
three sets of findings, three voices, three publication-ready outputs.

**Core principle:** Different epistemic positions see different things. A
research question filtered through a cognitive-workflow architecture
produces different findings than the same question filtered through a
frame-critical, sovereignty-aware architecture. Running both in parallel
and analysing their convergence and divergence reveals what neither
pipeline produces alone.

**Three components:**

1. **CRIA-Cognitive** — ten cognitive-role channels optimised for evidence
   aggregation under disciplined workflow
2. **CRIA-Epistemic** — ten epistemic-mode channels optimised for
   frame-critical, refusal-aware, civilisationally-anchored reading
3. **CRIA-Convergent** — comparison engine operating above both, producing
   findings about the disagreement itself

Each component has its own Layer 3 meta-cognitive learning system. The
unified orchestrator runs all three in parallel and renders findings in
three voices for three different audiences.

## 2. CRIA-Cognitive Architecture

### 2.1 Purpose

Optimised for *converging on findings under disciplined workflow*. Strong
for empirical questions where evidence aggregation across credentialed
sources produces actionable answers.

### 2.2 Ten Cognitive-Role Channels

1. **Scoping & Ontology** — defines research boundaries and entities
2. **Evidence Acquisition** — searches academic databases
3. **Contradiction & Anomaly Detection** — flags conflicts and outliers
4. **Synthesis & Abstraction** — integrates findings into coherent picture
5. **Causal & Relational Mapping** — infers causal dependencies
6. **Critic & Falsification** — attempts to disprove hypotheses
7. **Serendipity & Discovery** — finds non-obvious connections
8. **Quality Control** — assesses source credibility and methodology
9. **Cultural Context** — assesses cultural scope and validity
10. **Process Steering** — reflects on process and reallocates

### 2.3 Connector Ecology — Equal Weight to CRIA-Epistemic

CRIA-Cognitive uses ~35-40 connectors focused on *evidence-synthesis
territory* (different ecology from CRIA-Epistemic but equal scale):

**Mainstream academic (5):** Semantic Scholar, OpenAlex, PubMed, arXiv,
Crossref

**Quantitative empirical specialist (8):** Scopus (paid, phase 2),
Dimensions, JSTOR, Google Scholar API, Zenodo, figshare, ICPSR, Harvard
Dataverse

**Health and clinical (6):** UpToDate, ClinicalTrials.gov, Cochrane Library,
Europe PMC, MEDLINE, NICE Evidence Search

**Policy and economic data (6):** World Bank Open Data, OECD.stat, FRED,
data.gov.au, Eurostat, UN Statistics

**Grey literature empirical (4):** OpenGrey, ProQuest Dissertations, Open
Science Framework, ReplicationWiki

**Replication and meta-analysis (4):** Cochrane reviews, Campbell
Collaboration, ReplicationMarkets, AllTrials registry

**Position-privilege:** All credentialed-research positioned. Single epistemic
position deliberately — this is what defines CRIA-Cognitive's territory.

### 2.4 Layer 2 — Meta-Layer

Single meta-layer doing:
- Novelty scoring (1-5 threshold filter on findings)
- Hidden cross-connection finder (one synthesis pass across channels)
- Iteration steering (continue/stop recommendations)

### 2.5 Layer 3 — Meta-Cognitive Learning

CRIA-Cognitive learns *which general pattern-detection archetypes work
for which research contexts*. Strategies:

- cross_domain_analogy_mapping
- residual_anomaly_clustering
- absence_as_signal
- isomorphic_graph_mismatch
- hidden_moderator_chain
- boundary_condition_inversion
- semantic_drift_bridge
- unused_constraint_exploitation
- temporal_sequential_echo
- meta_pattern_of_channel_biases

Heuristic: novelty + confidence + length (+ user feedback when available).
Stagnation recovery: random parameter mutation across strategies.

### 2.6 Hofstadter Protections (NEW — parity with CRIA-Epistemic)

Added discipline module operating after Layer 3, before final output:

- **Strange Loop Validator for evidence convergence** — detects when the
  pipeline reformulates the same finding without converging on action
- **Gödelian gate** — forces honest caveat language when high-confidence
  claims can't be verified from evidence base
- **Substrate logic for empirical contradictions** — distinguishes data
  problems from methodological problems when studies contradict
- **Eliza Effect for convergence** — checks whether agreeing sources are
  truly independent or all citing the same original

Both pipelines now have equivalent epistemological rigor at validation layer.

### 2.7 Findings Schema

CRIA-Cognitive findings include:
- content, source_channel, confidence, evidence
- evidence_tier (T1/T2/T3)
- modality (knowledge/belief)
- contradiction_flags, novelty_score
- finding_id

## 3. CRIA-Epistemic Architecture

### 3.1 Purpose

Optimised for *excavating frames, reading indexically and iconically,
honouring refusal traditions, civilisational anchoring*. Strong for
foundational questions where the framing itself is contested.

### 3.2 Ten Epistemic-Mode Channels

1. **Empirical / Quantitative** — numerical evidence, statistical
   methodology, effect sizes, replicated findings
2. **Phenomenological / Qualitative** — lived experience, narrative
   accounts, ethnographic depth
3. **Historical / Archaeological** — frame archaeology, longitudinal
   trajectories, frame extinction
4. **Philosophical / Theoretical** — apparatus development, philosophy
   proper, theoretical traditions
5. **Critical / Counter-corpus** — dissenting, decolonial, critical-AI,
   structurally marginalised perspectives
6. **Civilisational / Systemic** — long timescales, multi-society, post-AI
   meaning, Four Requirements
7. **Cross-cultural / Comparative** — Buddhist, Ubuntu, Confucian,
   Indigenous-relational framings
8. **Computational / Modelling / Simulation** — model-driven inference,
   Atlan, Schelling, Hofstadter Copycat
9. **Adversarial / Falsificationist** — steel-mans counter-positions,
   sustained adversarial reasoning
10. **Experimental / Wildcard** — Atlan noise principle, codelets,
    Slippability tracking

### 3.3 Connector Ecology — 40 Connectors

**Mainstream academic (5):** OpenAlex, arXiv, Crossref, Semantic Scholar,
PubMed (deliberate overlap with CRIA-Cognitive for triangulation)

**Theoretical-tradition specialist (7):** PhilPapers, PhilArchive, Stanford
Encyclopedia of Philosophy, Internet Encyclopedia of Philosophy,
Constructivist Foundations, Cybernetics and Human Knowing, nLab

**Critical / counter-corpus (8):** Big Data & Society, Indigenous AI Protocol,
AlterNative, Decolonization: Indigeneity Education and Society, Settler
Colonial Studies, Social Studies of Science, Science Technology & Human
Values, Hypatia

**Indigenous scholarship and sovereignty (8):** AIATSIS, Lowitja Institute,
NACCHO, NATSILS, Local Contexts, Te Mana Raraunga, Maiam nayri Wingara,
First Nations Media Australia (6 partnership-gated, 2 active)

**Australian institutional (7):** AustLII, WorldLII, data.gov.au, ARDC,
Productivity Commission CTG, NIAA, AHRC

**International institutional (5):** UN PFII, UNDRIP, World Bank, ILO,
UNESCO

**Position-privilege diversity:** All seven positions represented
(state_admin, credentialed_research, community_curated,
indigenous_scholarship, theoretical_tradition, advocacy, grey_practitioner).

### 3.4 Layer 2 — Two-Stream Metagent

**Academic stream:** Formal apparatus, citations, position-privilege
accounting, convergence/divergence with falsification requirement,
refusal-as-finding when sovereign sources flag it.

**Experimental stream:** Juniper-influenced — Atlan, von Foerster,
Maturana-Varela, Bateson, Hofstadter, Eco abductive economy, Peirce
icon/index/symbol, Schelling salience.

Both streams apply meta-archetype queries (cross-domain analogy, frame
extinction, absence-as-signal, refusal precedence, etc.). Findings
surviving both streams are high-confidence.

### 3.5 Layer 3 — Meta-Cognitive Learning

CRIA-Epistemic learns *which v4-distinctive frame-critical strategies
earn their keep*. Strategies:

- position_privilege_rebalancing
- dissonance_budget_calibration
- refusal_precedence_detection
- frame_extinction_tracking (longitudinal logs)
- sovereign_aggregation_audit
- strange_loop_validation_tuning
- two_voice_fidelity_check

Heuristic: frame-distinctness + position-privilege diversity + Hofstadter
actionable-signal count. Stagnation recovery: raise dissonance budget
+0.15 (capped 0.80), re-weight to counter-corpus.

### 3.6 Hofstadter Protections

Already implemented:
- Strange Loop Validator (level-shift detection)
- Gödelian reset (epistemic honesty when claims unprovable within corpus)
- Substrate logic (contradictions signal ontology expansion vs error)
- Eliza Effect warning (syntactic vs semantic wins)
- Slippability tracking (boundary-breaking metadata for wildcard)

### 3.7 Findings Schema — Extended

CRIA-Epistemic findings include all CRIA-Cognitive fields PLUS:
- position_privileged
- dissonance_role
- evidence_tier
- sovereign_aggregation_check
- partnership_gated
- refusal_signal
- frame_inventory_match
- reading_mode (symbolic/indexical/iconic)
- slippability_metadata
- strange_loop_check
- substrate_signal

## 4. CRIA-Convergent — The Comparison Engine

### 4.1 Purpose

A *third research instrument* operating above both pipelines. Not a
commentary layer but a genuine pattern-detection engine that finds what
becomes visible only when two architectures are read in conversation
with each other.

### 4.2 Five Cross-Pipeline Analytical Channels

1. **Convergence Topology** — where findings from different epistemic
   positions cluster around the same phenomenon, suggesting reality
   independent of frame
2. **Divergence Anatomy** — where pipelines disagree and the disagreement
   reveals presuppositions underlying the question itself
3. **Absence Mapping** — what neither pipeline found but the gap between
   them suggests should exist
4. **Frame Collision** — where CRIA-Cognitive's frame and CRIA-Epistemic's
   frame meet head-on, revealing ontological assumptions neither
   pipeline questions independently
5. **Evidence Ecology Comparison** — what each pipeline's databases reveal
   versus what they miss; what the difference tells about what's knowable
   versus what's been made unknowable

### 4.3 Layer 3 — Comparison Meta-Cognitive Learning

CRIA-Convergent learns *which cross-pipeline analytical moves are most
productive*. Strategies:

- convergence_confidence_scaling — when both agree, how much does
  confidence increase (sub-linearly? logarithmically?)
- disagreement_diagnosis — epistemic, frame-based, or methodological?
- blind_spot_mapping — what does one pipeline see that the other doesn't?
- frame_archaeology_via_disagreement — what does the shape of
  disagreement reveal about presuppositions?
- evidence_ecology_triangulation — what do source differences reveal?

Heuristic: pattern is validated only if *both* individual pipelines'
Layer 3 systems confirm the comparison finding as distinctive and
actionable. Higher bar than either pipeline alone.

### 4.4 Output

CRIA-Convergent produces its own findings (as Convergence Findings) plus
a meta-synthesis that references both pipelines without averaging them.
The output answers questions like:

- What can we say with confidence given both readings?
- Where does the disagreement reveal something about the question itself?
- What presuppositions does the question carry?
- Which traditions would refuse the premise?
- How should a reader receive findings given their position-privilege
  context?

## 5. Three-Voice Rendering

Each research run produces findings in three voices, oriented toward
different audiences and purposes.

### 5.1 Academic Voice

**Optimises for:** Formal rigor, citation discipline, evidence-tier
transparency, position-privilege accounting, methodological soundness.

**Audience:** Peer-reviewed publication, specialist scholarly community.

**Structure:** Abstract, methodology, findings with citations,
discussion, limitations, conclusion. Position-privilege distribution
explicit. Refusal signals foregrounded as primary findings where
applicable.

**Quality gates:** Citation accuracy, falsification conditions stated,
observer note present, sovereign-source non-aggregation honoured.

### 5.2 Editorial Voice

**Optimises for:** Cool, contemporary, journalistic clarity for
educated general readers without sacrificing rigor.

**Audience:** Trade publications, quality magazines (Atlantic, Wired,
Aeon-style), Substack, podcasts, social media threads, intelligent
public discourse.

**Structure:** Lead paragraph that lands the central finding, context
that explains why it matters, the underlying reasoning made accessible,
acknowledgment of complexity without losing momentum, ending that
positions the reader to think further.

**Function:**
- Translates academic findings for educated general readers
- Bridges specialist findings between fields (helps a clinician
  understand philosophy findings; helps a policy analyst understand
  computational findings)
- Decodes academically obscure output when this is useful
- Helps researchers using CRIA in fields outside their specialist
  knowledge produce comprehensible results

**Quality gates:** Accuracy maintained from academic version, no
specialist jargon without explanation, claims appropriately hedged,
voice consistent (cool, contemporary, professional-journalistic).

### 5.3 Practitioner Voice

**Optimises for:** Decision-orientation, actionability, transparent
confidence levels, alternative-framework awareness.

**Audience:** Clinicians, policy makers, community organisers,
consultants, anyone who needs to *use* findings rather than publish
them.

**Structure:** What this means for your work, where evidence is strong
enough to implement with confidence, where to pilot-test, what
alternative frameworks suggest, what assumptions underlie any
recommendations, implementation considerations, who to consult before
acting.

**Function:**
- Helps people without academic background but with serious purpose get
  understandable, usable results
- Makes implementation safe by surfacing what would otherwise stay
  implicit (assumptions, frame-contingencies, alternative perspectives)
- Lets the research *do something* in the world rather than just
  inform it

**Quality gates:** Actionable specificity, confidence calibrated to
evidence strength, alternative frameworks acknowledged, ethical
considerations surfaced, partnership requirements named where
relevant.

### 5.4 Voice Selection

Default: all three voices rendered, user selects which to display or
download. Single-voice runs available when only one is needed (saves
LLM cost).

## 6. Publication Guidance Engine

### 6.1 Purpose

Built-in venue suggestions based on what the pipelines actually produced.
Reads metadata from each pipeline and recommends 2-3 publication venues
per output, with structure suggestions for each.

### 6.2 Metadata Inputs

Each pipeline produces metadata used by the guidance engine:

- Position-privilege distribution
- Evidence-tier composition (T1/T2/T3 percentages)
- Dissonance budget actually integrated
- Refusal signals encountered
- Frame-extinction observations
- Layer 3 performance report
- Channel firing pattern (which channels produced strongest signals)

### 6.3 Venue Mapping Logic

**For CRIA-Cognitive output:**

- Heavy credentialed-research position-privilege + T1-dominant + low
  refusal = empirical methodology venues (Research Synthesis Methods,
  Systematic Reviews, Evidence Synthesis & Implementation)
- Clinical/health domain = applied medicine (JMIR, BMC Medicine,
  domain-specific journals)
- Education/social science = education or policy (Educational Research
  Review, Evidence & Policy)

**For CRIA-Epistemic output:**

- High refusal-signal + strong frame-extinction = epistemology
  (Episteme, Philosophy of Science, BJPS)
- Heavy Indigenous-scholarship or community-curated = decolonial venues
  (Decolonization: Indigeneity Education and Society, AlterNative,
  Settler Colonial Studies)
- High dissonance-budget + counter-corpus dominant = critical theory/STS
  (Science Technology & Human Values, Social Studies of Science,
  Critical Sociology)
- Civilisational-anchoring strong = futures/policy-philosophy (Futures,
  AI & Society, Theory Culture & Society)

**For CRIA-Convergent output:**

- Both pipelines converged with high confidence = applied methodology
  in the question's domain
- Sharp divergence revealing frame-contingency = epistemology/methodology
  (Episteme, Philosophy of Science, Social Epistemology)
- Disagreement reveals invisible presuppositions = specialised journals
  in domain plus methodology venues

### 6.4 Output

```
CRIA-Cognitive findings suggest publication in:
  1. [Primary venue with reasoning]
  2. [Secondary venue]
  3. [Tertiary venue]

CRIA-Epistemic findings suggest publication in:
  1. [Primary venue]
  2. [Secondary venue]
  3. [Tertiary venue]

CRIA-Convergent dual-pipeline analysis suggests publication in:
  1. [Primary venue]
  2. [Secondary venue]
  3. [Tertiary venue]

Combined-pipeline papers (where findings work together):
  - [Suggested combination with target venue]
```

### 6.5 Three-Paper Strategy

The unified architecture supports three publishable papers from one
research run:

1. **CRIA-Cognitive paper** — empirical methodology, evidence synthesis
   contribution
2. **CRIA-Epistemic paper** — frame-critical, decolonial, theoretical
   contribution
3. **CRIA-Convergent paper** — methodological innovation, dual-pipeline
   methodology as novel research practice

Sequence: get each pipeline strong independently, write single-pipeline
papers first, *then* the dual-pipeline paper builds on demonstrated
excellence from both.

## 7. Connector Expansion Discipline

### 7.1 Current State

Both pipelines use only free-tier, open-access sources. This is a
*legitimate epistemic choice* — what's available without institutional
gatekeeping. Worth publishing with this constraint visible.

### 7.2 Phase 2 Expansion (Future)

When bandwidth permits, add 8-12 high-quality paid sources per pipeline,
chosen deliberately for what they uniquely provide:

**For CRIA-Cognitive:** Scopus or Dimensions (citation networks,
non-English coverage), JSTOR institutional, specialist health database.

**For CRIA-Epistemic:** PhilPapers paid journal access, decolonial
journal archives, Indigenous-scholarship specialist databases via
partnership-gated arrangements.

### 7.3 The Noise Balance — Connector Gate

Before adding any new connector, evaluation gate:
- Does it materially change what the pipeline sees?
- Does it surface findings the existing connectors don't?
- Or does it just add redundancy and false-convergence noise?

Test on one research question. If new source surfaces findings neither
pipeline found before, it earns its connector. If it just reproduces
existing findings in slightly different language, it doesn't.

This prevents bloat while remaining intellectually honest about
accessible sources.

## 8. Unified Orchestrator Architecture

### 8.1 Entry Point

Single `/research` endpoint accepting:
- Research question (required)
- Observer note (recommended)
- Mode: cognitive_only | epistemic_only | convergent (default)
- Voice: academic | editorial | practitioner | all (default)
- Dissonance budget (CRIA-Epistemic parameter, 0.0-1.0)
- Profile: general_scholarship | partnership_sensitive
- Max iterations (1-3, default 2)

### 8.2 Execution Flow

1. **Parse request** — validate parameters, check API key availability
2. **Spawn pipelines in parallel** — `asyncio.gather` runs:
   - CRIA-Cognitive (10 channels + Layer 2 + Layer 3 + Hofstadter)
   - CRIA-Epistemic (10 channels + 2-stream metagent + Hofstadter +
     Layer 3)
3. **Wait for both** — with timeout handling (partial results returned
   if one fails)
4. **Run CRIA-Convergent** — comparison engine reads both outputs,
   produces 5-channel analysis, runs its own Layer 3
5. **Render three voices** — academic, editorial, practitioner versions
   of all three outputs
6. **Generate publication guidance** — venue suggestions per pipeline
   plus combined-paper opportunities
7. **Return unified result** — structured JSON with all findings, all
   voices, comparison analysis, publication roadmap

### 8.3 Cost Management

Full convergent run with all three voices: ~$1.50-$3.00 per query at
2 iterations with claude-sonnet-4-5. User can reduce by:
- Single-pipeline mode (CRIA-Cognitive only or CRIA-Epistemic only)
- Single-voice rendering
- Lower iteration count

Default cost-conscious mode: 1 iteration, all three voices, full
convergent. ~$0.75-$1.50 per query.

## 9. Unified Dashboard

### 9.1 Design Principles

- Cool, contemporary, professional aesthetic
- Clear architectural transparency (user sees what each pipeline did)
- Mode switching without page reload
- Help and tooltips throughout for users new to the methodology
- Three-voice output displayed in tabs (academic | editorial |
  practitioner)
- Publication guidance panel always visible after run completes

### 9.2 Layout

**Header:** CRIA branding, mode selector (cognitive_only | epistemic_only
| convergent default), help button

**Query panel:** Research question textarea, observer note input,
parameter controls (dissonance budget, profile, voice, iterations)
with tooltips explaining each

**Execution panel:** Loading spinner with progress indicators per
pipeline (CRIA-Cognitive running... CRIA-Epistemic running...
CRIA-Convergent comparing...)

**Results panel:** Three sections (CRIA-Cognitive | CRIA-Epistemic |
CRIA-Convergent) each expandable, with tabbed voice selection inside

**Publication panel:** Venue suggestions per output, combined-paper
opportunities, structure recommendations

**Findings detail:** All findings tagged with metadata (position-privilege,
evidence-tier, dissonance-role, refusal-signal, reading-mode,
partnership-gated)

### 9.3 Help System

Inline tooltips for:
- Each parameter (what it does, recommended ranges)
- Each pipeline (what it's optimised for, when to use)
- Each voice (audience, purpose)
- Each metadata tag (what it means)
- Each Layer 3 strategy (what it's learning)
- Publication guidance (how venue matching works)

Glossary panel accessible from header explaining all CRIA-specific
terminology.

## 10. Build Sequence

### Phase 1 — Unified Codebase (NEXT)
1. Build unified `cria_unified.py` containing both pipelines + comparison
   engine + orchestrator + three-voice rendering + publication guidance
2. Build unified dashboard with all features above
3. Validate Python syntax and rough functional testing

### Phase 2 — Replit Deployment
1. Deploy to Replit with Anthropic API integration
2. First test query with all three pipelines firing
3. Verify three-voice rendering works correctly
4. Verify publication guidance produces sensible suggestions

### Phase 3 — Independent Validation
1. Run CRIA-Cognitive standalone on a research question; document
   strengths/limits
2. Run CRIA-Epistemic standalone on same question; document
   strengths/limits
3. Compare independently-produced findings to convergent-mode findings

### Phase 4 — First Three-Paper Run
1. Choose substantive research question (likely from What Remains book
   work or HUM therapeutic media work)
2. Run full convergent mode
3. Draft three papers (cognitive, epistemic, convergent) targeting
   suggested venues
4. Submit for peer review

### Phase 5 — Connector Expansion
1. After Phase 4 demonstrates value, evaluate paid sources
2. Add 8-12 paid connectors per pipeline through evaluation gates
3. Re-run Phase 4 questions to assess what changes

## 12. Disciplines Maintained

- **Partnership-gating preserved**: Indigenous-scholarship sources
  catalogued but inactive until partnership conditions met (Troy
  gating)
- **Sovereign-source non-aggregation**: Indigenous scholarship appears
  in result sets but never aggregated for triangulation as if
  equivalent to other frames
- **Refusal as first-class output**: when sovereign sources flag
  refusal, the metagent foregrounds refusal rather than synthesising
  past it
- **No fabrication**: all voices instructed to name gaps rather than
  invent content; do not invent citations
- **Honest staging**: free-tier sources first, paid expansion through
  evaluation gates
- **Observer note recommended**: declares the researcher's position;
  required for partnership-sensitive profile
