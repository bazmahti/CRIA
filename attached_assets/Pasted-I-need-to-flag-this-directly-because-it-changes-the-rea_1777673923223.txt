The diagnostic
There are a few possibilities for what's happening, and I want to be straight about which seems most likely before recommending a fix.
Possibility one — the runtime is producing one paper and rendering it three times for display. This would mean the dual-pipeline architecture isn't yet differentiating between Cognitive, Epistemic, and Convergent outputs. All three pipelines are producing the same text, or the rendering layer is collapsing three pipeline outputs into one synthesised document and showing it three times. The "Show N channel findings" footer is the only differentiator and it varies (24, 13, 7) which suggests the underlying channels did fire and produce distinct findings — but those findings aren't being incorporated into the rendered paper text. They're sitting in metadata that the paper rendering is ignoring.
Possibility two — the runtime is producing three distinct papers but you've pasted the same one three times by accident. Possible on a phone where copy-paste is fiddly. Worth checking before drawing architectural conclusions. If the dashboard has three tabs or three download links, it's possible the second and third pasted in identically because the wrong source got copied each time.
Possibility three — the runtime is producing three pipelines' worth of findings but rendering them through a shared document template that doesn't yet differentiate by pipeline. This would be an architectural issue at the rendering layer specifically — the pipelines are working, the findings are being tagged, but the paper-generation step is feeding the same template the same input regardless of which pipeline it's rendering for.
I lean toward possibility three being most likely, with possibility two as a secondary check worth doing. The variable channel-finding counts (24, 13, 7) suggest the underlying experimentation is producing differentiated output. The identical body text suggests the paper-rendering step is the bottleneck.
